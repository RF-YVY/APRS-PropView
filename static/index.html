📖 APRS PropView — Help & User Guide

Overview

APRS PropView is a real-time APRS digipeater and Internet Gateway (IGate) application focused on visualizing VHF propagation conditions. It decodes APRS packets from RF (via a TNC) and from the APRS-IS network, plots stations on a live map, draws signal paths, and scores current propagation quality.

🗺️ Map

The central map shows your station (blue dot), RF stations (red border), and APRS-IS stations (blue border). Propagation lines animate from each RF station toward your position, color-coded by distance:

  • Red — < 31 mi (50 km)
  • Orange — 31–62 mi (50–100 km)
  • Green — 62–124 mi (100–200 km)
  • Purple — > 124 mi (200 km) — DX!

Small white dots along a path indicate digipeater hops.

🎛️ Map Controls

📍Center Map — Re-center on your station.
📏Toggle Lines — Show/hide propagation path lines.
🔴Toggle RF — Show/hide RF station markers.
🔵Toggle APRS-IS — Show/hide Internet-heard stations.
📌Pick Location — Click on the map to set your station coordinates.
🌙 / ☀️Theme — Toggle dark/light map tiles.
🏷️Callsign Labels — Show/hide callsign text above each station icon.
🔎Auto-Fit — Automatically zoom to fit all visible stations. Disabled when you manually pan or zoom; click again to re-enable.
⏱️Line Time Filter — Only show path lines from stations heard within the selected window (1h, 6h, 12h, 24h, or all time).
🏷️ Types ▾Type Filter — Check/uncheck station categories (Digipeater, Weather, House, Car, etc.) to control which types appear on the map.

All map toggle states, zoom level, position, and filter selections are saved automatically and persist across restarts.

📻 RF Stations & 🌐 APRS-IS Tabs

Scrollable lists of stations heard on RF or via the APRS-IS network. Each entry shows the station's APRS icon, callsign, distance, heading, last heard time, and packet count. Use the filter dropdowns to narrow by time window, distance range, or station type.

📦 Packets Tab

A live feed of decoded APRS packets. Each entry shows the source (RF or IS), timestamp, raw packet contents, and decoded fields. Use the source filter dropdown or the Clear button to manage the list.

💬 Messages Tab

Send and receive APRS messages. Type a destination callsign and message text (max 67 characters), then click Send. Incoming messages appear with ack/rej status indicators. Click any received message's callsign to auto-fill the To field for a quick reply.

Message notification channels (Discord, Email, SMS) are configured separately from band-opening alert channels in Settings → Alerts.

📊 Propagation Tab

Real-time propagation scoring with stat cards: RF stations (1h/6h/24h), max distance, average distance, and APRS-IS count. Includes a distance distribution histogram and a 24-hour propagation history chart showing station count and max distance over time.

📊 Analytics Tab

Advanced analysis subtabs:

  • Longest Paths — Leaderboard of the greatest distances reached.
  • Heatmap — Time-of-day vs. distance heatmap showing when propagation peaks.
  • Station Reliability — Which stations are heard most consistently.
  • Best Times — Hourly chart showing the best time of day for propagation.
  • Alerts — History of band-opening alerts triggered by the system.

⚙️ Settings

All settings are saved to config.toml next to the executable. Some changes (callsign, TNC, APRS-IS) require a restart; the app will notify you.

Station

Your callsign, SSID, latitude/longitude (or use the 📌 map picker), APRS symbol, status comment, beacon interval (minutes), and beacon path (e.g. WIDE1-1).

Digipeater

Enable/disable the digipeater. Configure which aliases to respond to (WIDE1-1, WIDE2-1) and the de-duplication window to prevent re-digipeating the same packet.

IGate

Enable/disable Internet gateway. Choose RF→IS (gate heard RF packets to APRS-IS), IS→RF (gate APRS-IS packets to local RF), or both.

APRS-IS

Server/port (default: rotate.aprs2.net:14580), your APRS-IS passcode, range filter in miles (auto-generates the r/lat/lon/km server filter), and any extra filter expressions (e.g. b/CALL).

KISS Serial / KISS TCP

Connect to your TNC via serial port (COM port + baud rate) or TCP (host + port). Enable one or both.

Web Interface

Host/Port — Bind address and port for the web UI (default 127.0.0.1:14501). Use 0.0.0.0 to allow network access.

Font — Choose a display font for the entire UI.

Ghost After (min) — Stations not heard within this time will fade (pulsing dashed border, reduced opacity). Set to 0 to disable ghosting.

Expire After (min) — Stations not heard within this time are completely removed from the map. Set to 0 to never auto-expire. Ghost time should be shorter than expiry time so stations fade before disappearing.

Tracking

Max Station Age — How long (minutes) to keep a station in the database before cleanup removes it. Cleanup Interval — How often (minutes) to run the cleanup process.

Alerts — Band Opening Channels

Trigger an alert when propagation exceeds thresholds (min RF stations + min distance). Cooldown prevents repeated alerts. Quiet hours suppress alerts during specified times. Channels: Discord webhook, Email (SMTP), and SMS via email gateway.

Alerts — Message Notification Channels

Separate channel toggles for forwarding incoming APRS messages to Discord, Email, or SMS — independent from band-opening channels.

Weather

Enable weather display with an NWS location code (e.g. a US grid point or zone). Severe weather alerts within the configured range will appear as banners above the map. Refresh interval controls how often conditions are fetched.

📱 Mobile View

Access /mobile from a phone browser (e.g. via Tailscale) for a touch-optimized interface with Messages, Map, Stations, Propagation, and About tabs.

⌨️ Tips

  • Distance units (mi/km) can be toggled via the button in the map legend — the setting persists across sessions.
  • All map UI preferences (toggles, filters, zoom, position, theme) are saved automatically to your browser and restored on next launch.
  • Ghost + Expire work together: set Ghost to 60 min and Expire to 180 min, for example, so stations fade at 1 hour and vanish at 3 hours.
  • Auto-Fit zoom (🔎) will keep all visible stations in view and zoom back in as stations expire. It pauses when you manually pan or zoom — click the button again to re-enable.
  • The propagation score is calculated from RF station count, maximum distance, and average distance within the last hour.
